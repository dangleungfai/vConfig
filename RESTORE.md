# vConfig 代码恢复指南

本文说明如何从 Git 备份标签或提交恢复 vConfig 代码版本。

## 一键恢复当前备份版本

如果后续出现问题需要恢复到当前备份版本，请按以下步骤操作：

### 方法一：使用备份标签恢复（推荐）

```bash
# 1. 查看所有备份标签
git tag -l

# 2. 恢复到最新的备份标签（例如：backup-20250206-123000）
git checkout backup-20250206-123000

# 或者恢复到指定的备份标签
git checkout <备份标签名>
```

### 方法二：使用提交哈希恢复

```bash
# 1. 查看提交历史
git log --oneline

# 2. 恢复到指定的提交
git checkout <提交哈希>
```

### 方法三：创建恢复分支（推荐用于生产环境）

```bash
# 1. 基于备份标签创建恢复分支
git checkout -b restore-backup backup-20250206-123000

# 2. 如果需要合并回主分支
git checkout main  # 或 master
git merge restore-backup
```

## 注意事项

1. **恢复前请先备份当前代码**：
   ```bash
   cp -r . ../vconfig_backup_$(date +%Y%m%d)
   ```

2. **恢复后需要重启服务**：
   ```bash
   # 停止当前服务
   # 然后重新启动
   python3 app.py
   ```

3. **数据库变更**：如果恢复后数据库结构有变化，可能需要运行迁移或重新初始化。

## 创建新备份

运行备份脚本创建新的备份点：

```bash
chmod +x backup.sh
./backup.sh
```
